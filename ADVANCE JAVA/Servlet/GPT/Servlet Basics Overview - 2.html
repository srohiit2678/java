1. Introduction to Servlets

A Servlet is a Java class used to extend the capabilities of a web server. It handles HTTP requests and responses, making it a key component in Java web applications. Servlets run on a Servlet Container (like Apache Tomcat) and help generate dynamic web content.

How Servlets Work

  1. Client Sends a Request → Browser sends an HTTP request to the web server.
  2. Server Passes the Request to Servlet Container → The container checks for a matching servlet.
  3. Servlet Processes the Request → It executes Java code, interacts with a database, and prepares a response.
  4. Response Sent Back to Client → The servlet sends an HTTP response (HTML, JSON, etc.).

2. Servlet Lifecycle

A servlet goes through the following phases:

  • Loading and Instantiation → The servlet class is loaded and an instance is created.
  • Initialization (init() method) → The servlet is initialized and ready to handle requests.
  • Handling Requests (service() method) → For each request, service() calls either doGet() or doPost().
  • Destruction (destroy() method) → Before shutting down, the servlet releases resources.

3. Writing Your First Servlet

Steps to Create a Simple Servlet

  1. Create a Java Class that extends HttpServlet.
  2. Override doGet() or doPost() to handle requests.
  3. Configure web.xml or use @WebServlet annotation.
  4. Deploy on a Servlet Container (Tomcat, GlassFish, etc.).

Example: Basic Servlet

java
import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/hello")
public class HelloServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        response.setContentType("text/html");
        response.getWriter().println("<h1>Hello, Servlet!</h1>");
    }
}

Deploying the Servlet

  • Without Annotation → Configure web.xml
xml
<web-app>
    <servlet>
        <servlet-name>HelloServlet</servlet-name>
        <servlet-class>HelloServlet</servlet-class>
    </servlet>
    <servlet-mapping>
        <servlet-name>HelloServlet</servlet-name>
        <url-pattern>/hello</url-pattern>
    </servlet-mapping>
</web-app>
  • With Annotation → No need for web.xml.

2. How @WebServlet("/hello") Works Internally

When we use:

java

    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {

How It Works Internally

  1. The Servlet Container (Tomcat, GlassFish, etc.) scans for classes with @WebServlet annotation.
  2. It registers HelloServlet with the URL pattern /hello.
  3. When a request is sent to http://localhost:8080/myapp/hello, the container:
    • Creates a new HttpServletRequest and HttpServletResponse object.
    • Calls doGet() of HelloServlet.
    • Sends the response back to the browser.

3. Why @WebServlet Removes the Need for web.xml?

Before Java EE 6, servlets were configured using web.xml:

xml
<web-app>
    <servlet>
        <servlet-name>HelloServlet</servlet-name>
        <servlet-class>HelloServlet</servlet-class>
    </servlet>
    <servlet-mapping>
        <servlet-name>HelloServlet</servlet-name>
        <url-pattern>/hello</url-pattern>
    </servlet-mapping>
</web-app>

Problems with web.xml

  • It's verbose (requires more code).
  • Manual mapping increases errors.
  • Not developer-friendly.

Why @WebServlet is Better?

  • Less code → No need for XML configuration.
  • Annotations are easier to read and maintain.
  • Automatic scanning → The container automatically detects servlets.

Filters and Listeners in Servlets - Deep Dive

Filters and Listeners are powerful components in Java Servlets that help with request processing, security, logging, and application monitoring.

1. Filters in Servlets

1.1 What are Filters?

A Filter in Servlets is an interceptor that sits between the client request and the servlet. It can:

  • Pre-process the request (before it reaches the servlet).
  • Post-process the response (before sending it back to the client).
  • Modify request/response data.
  • Perform security checks, logging, compression, etc..

1.2 Filter Workflow

  1. Client sends a request → The filter intercepts it.
  2. Filter processes the request (e.g., authentication, logging).
  3. Request is passed to the servlet (if allowed).
  4. Servlet processes the request and generates a response.
  5. Filter modifies the response (if needed).
  6. Response is sent back to the client.

1.3 Example 1: Logging Filter

This filter logs all incoming requests before they reach the servlet.

java
import java.io.IOException;
import javax.servlet.*;
import javax.servlet.annotation.WebFilter;

@WebFilter("/*") // This applies the filter to all requests
public class LoggingFilter implements Filter {

    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {
        System.out.println("Request received at: " + new java.util.Date());
        chain.doFilter(request, response); // Continue to the servlet
    }
}

How It Works?

  • The filter runs before every servlet execution.
  • chain.doFilter(request, response); → Allows the request to pass to the next filter or servlet.
  • After servlet execution, it can modify the response.

1.4 Example 2: Authentication Filter

This filter checks if the user is logged in before allowing access to protected resources.

java
import java.io.IOException;
import javax.servlet.*;
import javax.servlet.annotation.WebFilter;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.http.HttpSession;

@WebFilter("/secure/*") // Applies to all URLs under /secure/
public class AuthenticationFilter implements Filter {

    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {

        HttpServletRequest req = (HttpServletRequest) request;
        HttpServletResponse res = (HttpServletResponse) response;
        HttpSession session = req.getSession(false);

        if (session == null || session.getAttribute("user") == null) {
            res.sendRedirect("/login.jsp"); // Redirect to login page
        } else {
            chain.doFilter(request, response); // User is logged in, allow access
        }
    }
}

How It Works?

  • Before the servlet runs, it checks if the user session exists.
  • If not logged in, it redirects to login.jsp.
  • If logged in, the request proceeds to the servlet.

1.5 Filter Lifecycle

A filter has three lifecycle methods:

  1. init(FilterConfig config) → Runs once when the filter is loaded.
  2. doFilter(ServletRequest request, ServletResponse response, FilterChain chain) → Runs for every request.
  3. destroy() → Runs when the filter is removed from the server.
java
public void init(FilterConfig config) throws ServletException {
    System.out.println("Filter initialized!");
}
public void destroy() {
    System.out.println("Filter destroyed!");
}

2. Listeners in Servlets

2.1 What are Listeners?

A Listener is a special component that listens for specific events in the web application. It tracks:

  • Session creation and destruction (HttpSessionListener)
  • Application startup and shutdown (ServletContextListener)
  • Attribute changes in session/application scope (HttpSessionAttributeListener)

2.2 Listener Workflow

  1. An event occurs (e.g., user logs in, session ends, app shuts down).
  2. Listener detects the event.
  3. It executes code to handle the event.

2.3 Example 1: Tracking Active Users with HttpSessionListener

This listener keeps track of how many users are logged in.

java
import javax.servlet.annotation.WebListener;
import javax.servlet.http.HttpSessionEvent;
import javax.servlet.http.HttpSessionListener;

@WebListener
public class ActiveUserListener implements HttpSessionListener {
    private static int activeUsers = 0;

    public void sessionCreated(HttpSessionEvent event) {
        activeUsers++;
        System.out.println("User logged in. Active users: " + activeUsers);
    }

    public void sessionDestroyed(HttpSessionEvent event) {
        activeUsers--;
        System.out.println("User logged out. Active users: " + activeUsers);
    }
}

How It Works?

  • When a new session is created (sessionCreated()), the active user count increases.
  • When a session is destroyed (sessionDestroyed()), the count decreases.

This listener executes code when the web application starts or stops.

java
import javax.servlet.ServletContextEvent;
import javax.servlet.ServletContextListener;

@WebListener

        System.out.println("Application shutting down!");
    }

How It Works?

  • When the server starts, it prints "Application started!".
  • When the server shuts down, it prints "Application shutting down!".

2.5 Example 3: Tracking Attribute Changes (HttpSessionAttributeListener)

This listener monitors attribute changes in a session.

java
import javax.servlet.annotation.WebListener;
import javax.servlet.http.HttpSessionAttributeListener;

@WebListener
public class SessionAttributeListener implements HttpSessionAttributeListener {

    public void attributeAdded(HttpSessionBindingEvent event) {

    }

How It Works?

  • When an attribute is added to the session: "Attribute added: username = JohnDoe".
  • When an attribute is removed: "Attribute removed: username".
  • When an attribute is changed: "Attribute replaced: username".

✅ Filters

  • Pre/Post-process requests before/after they reach servlets.
  • Common Use Cases: Logging, authentication, request modification.
  • Example: Logging filter, Authentication filter.

✅ Listeners

  • Track application and session events.

  • Example: HttpSessionListener for user tracking, ServletContextListener for startup actions.

Tracking Active Users with HttpSessionListener

What is HttpSessionListener?

HttpSessionListener is an interface in the Java Servlet API that listens for session creation and destruction events. It helps in tracking how many users are actively using the web application.

Whenever a new session is created, the listener detects it and increments the count of active users. Similarly, when a session is destroyed, the count is decremented.

🚀 Step-by-Step Execution

Let's break down the example code you asked about:

java
import javax.servlet.annotation.WebListener;
import javax.servlet.http.HttpSessionEvent;
import javax.servlet.http.HttpSessionListener;

@WebListener
public class ActiveUserListener implements HttpSessionListener {
    private static int activeUsers = 0;

    public void sessionCreated(HttpSessionEvent event) {
        activeUsers++; // Increment count when a session is created
        System.out.println("User logged in. Active users: " + activeUsers);
    }

    public void sessionDestroyed(HttpSessionEvent event) {
        activeUsers--; // Decrement count when a session is destroyed
        System.out.println("User logged out. Active users: " + activeUsers);
    }
}

🔍 How Does This Work?

  1. A new user visits the website → A new session is created.

    • sessionCreated(HttpSessionEvent event) gets triggered.
    • The activeUsers counter increases by 1.
    • A log message is printed:
      pgsql
      User logged in. Active users: 1

  2. Another user logs in → A new session is created.

    • The count goes up again:
      pgsql
      User logged in. Active users: 2

  3. A user logs out or their session expires → The session is destroyed.

    • sessionDestroyed(HttpSessionEvent event) is triggered.
    • The counter decreases by 1.
    • Log message:
      pgsql
      User logged out. Active users: 1

🛠 Where is This Used?

  • Show real-time online users in a dashboard.
  • Restrict access if too many users are logged in (e.g., limited licenses).
  • Track user engagement in analytics.

💡 How to Improve It?

  1. Store activeUsers in ServletContext

    • Instead of using a static variable, store it in ServletContext so it can be accessed globally.
    java
    public void sessionCreated(HttpSessionEvent event) {
    ServletContext context = event.getSession().getServletContext();
    Integer activeUsers = (Integer) context.getAttribute("activeUsers");
    if (activeUsers == null) activeUsers = 0;
    context.setAttribute("activeUsers", ++activeUsers);
}

public void sessionDestroyed(HttpSessionEvent event) {
    ServletContext context = event.getSession().getServletContext();
    Integer activeUsers = (Integer) context.getAttribute("activeUsers");
    if (activeUsers != null && activeUsers > 0) {
        context.setAttribute("activeUsers", --activeUsers);
    }
}

  2. Display the active users count on a webpage

    • Add a JSP page to show the live count:
    jsp
    <p>Active Users: <%= application.getAttribute("activeUsers") %></p>

  3. Handle session timeout properly

    • If users leave without logging out, their session may stay alive.
    • You can set session timeout in web.xml:
    xml
    <session-config>
    <session-timeout>15</session-timeout> <!-- Auto logout after 15 mins -->
</session-config>

🔚 Final Thoughts

HttpSessionListener is a simple but powerful tool for tracking active users in real-time. By combining it with ServletContext, JSP display, and session timeout handling, you can build an efficient user monitoring system.

}

Now that we are tracking active users with HttpSessionListener, let's display the live count on a webpage using JSP.

1️⃣ Modify ActiveUserListener.java to Store Count in ServletContext

Instead of using a static variable, we’ll store activeUsers in the ServletContext. This allows us to access it from a JSP page.

📝 Updated ActiveUserListener.java

java
import javax.servlet.annotation.WebListener;
import javax.servlet.http.HttpSessionEvent;
import javax.servlet.http.HttpSessionListener;
import javax.servlet.ServletContext;

@WebListener
public class ActiveUserListener implements HttpSessionListener {
    public void sessionCreated(HttpSessionEvent event) {
        ServletContext context = event.getSession().getServletContext();
        Integer activeUsers = (Integer) context.getAttribute("activeUsers");
        
        if (activeUsers == null) activeUsers = 0; // Initialize if null
        context.setAttribute("activeUsers", ++activeUsers); // Increment count
        
        System.out.println("User logged in. Active users: " + activeUsers);
    }

    public void sessionDestroyed(HttpSessionEvent event) {
        ServletContext context = event.getSession().getServletContext();
        Integer activeUsers = (Integer) context.getAttribute("activeUsers");
        
        if (activeUsers != null && activeUsers > 0) {
            context.setAttribute("activeUsers", --activeUsers); // Decrement count
        }
        
        System.out.println("User logged out. Active users: " + activeUsers);
    }
}

3️⃣ Testing the Application

🔹 How It Works:

  1. User 1 opens the website → A session is created, count increases to 1.
  2. User 2 logs in from another device → Count increases to 2.
  3. User 1 logs out or closes the browser → Count decreases to 1.
  4. User 2 also logs out → Count decreases to 0.

📌 You can see the real-time count by visiting http://localhost:8080/myapp/activeUsers.jsp in a browser.

4️⃣ Auto-Refreshing the Count (Optional)

If you want the count to update automatically without refreshing the page, modify activeUsers.jsp to include AJAX.

📝 Auto-Refresh with JavaScript (activeUsers.jsp)

jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8" %>
<!DOCTYPE html>
<html>
<head>
    <title>Active Users</title>
    <script>
        function updateActiveUsers() {
            var xhr = new XMLHttpRequest();
            xhr.open("GET", "activeUsers.jsp", true);
            xhr.onreadystatechange = function() {
                if (xhr.readyState == 4 && xhr.status == 200) {
                    document.getElementById("count-box").innerHTML = xhr.responseText;
                }
            };
            xhr.send();
        }
        setInterval(updateActiveUsers, 5000); // Refresh every 5 seconds
    </script>
    <style>
        body { font-family: Arial, sans-serif; text-align: center; margin-top: 50px; }
        h1 { color: #333; }
        .count-box { font-size: 24px; color: #007bff; margin-top: 20px; }
    </style>
</head>
<body>
    <h1>Live Active Users Count</h1>
    <div id="count-box" class="count-box">
        Active Users: <%= application.getAttribute("activeUsers") %>
    </div>
</body>
</html>

🔹 Now, the count updates every 5 seconds without a full-page refresh! 🚀
