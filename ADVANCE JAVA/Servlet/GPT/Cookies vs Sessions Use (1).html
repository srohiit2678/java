Detailed Answer: Cookie vs. Session

In web development, both cookies and sessions are used to store user-related data, but they work differently in terms of storage location, security, lifespan, and use cases.

1. Cookies

What is a Cookie?

A cookie is a small piece of data stored on the client-side (browser) that helps websites remember user-related information. It is sent with every HTTP request to the server.

Characteristics of Cookies:

  • Stored in the user's browser.
  • Limited in size (usually 4KB per cookie).
  • Can be set to expire after a specific duration.
  • Less secure (can be modified or stolen by attackers via XSS attacks).
  • Sent with every HTTP request, increasing request size.

When to Use Cookies?

Storing user preferences Theme selection, language settings, etc.
Remembering login credentials Like Remember Me functionality.
Tracking user behavior Like website analytics, ad tracking.
Maintaining cart data temporarily Some e-commerce sites use cookies to remember items for guest users.

How Cookies Work?

  1. The server sends an HTTP response with a Set-Cookie header.
  2. The browser stores the cookie.
  3. The browser automatically includes the cookie in future requests to the server.

Example:

java
// Set a cookie in a servlet
Cookie cookie = new Cookie("username", "john_doe");
cookie.setMaxAge(60 * 60 * 24); // 1 day expiration
response.addCookie(cookie);

// Retrieve cookie in a servlet
Cookie[] cookies = request.getCookies();
if (cookies != null) {
    for (Cookie c : cookies) {
        if (c.getName().equals("username")) {
            System.out.println("Welcome back, " + c.getValue());
        }
    }
}

2. Sessions

What is a Session?

A session is a server-side mechanism to store user-related data temporarily during their visit. A unique session ID is generated and usually stored in a cookie to track the session.

Characteristics of Sessions:

  • Stored on the server-side, making them more secure.
  • Larger storage capacity compared to cookies.
  • Automatically expires after a certain period of inactivity.
  • Slower than cookies (since it requires a server lookup).

When to Use Sessions?

User authentication Storing user login details securely.
Shopping carts Keeping track of selected items during checkout.
Temporary data storage Like form data before submission.
Sensitive data storage Sessions store confidential user information securely.

How Sessions Work?

  1. A user logs in or performs an action that creates a session.
  2. The server generates a unique session ID and stores user-related data.
  3. The session ID is sent to the browser (usually in a cookie).
  4. On subsequent requests, the session ID is used to retrieve stored data.

Example:

java
// Store session data
HttpSession session = request.getSession();
session.setAttribute("user", "john_doe");

// Retrieve session data
String username = (String) session.getAttribute("user");
System.out.println("Welcome back, " + username);

// Invalidate session (logout)
session.invalidate();

1 Create a Servlet to Fetch Cart Items from Database

We need a servlet (FetchCartServlet.java) that retrieves cart data dynamically when Sarah visits the site after logging out.

java
import java.io.IOException;
import java.sql.*;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import org.json.JSONArray;
import org.json.JSONObject;

@WebServlet("/fetchCart")
public class FetchCartServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        
        response.setContentType("application/json");
        response.setCharacterEncoding("UTF-8");

        String email = request.getParameter("email"); // Get email from request (optional)
        JSONArray cartItems = new JSONArray();

        if (email != null && !email.isEmpty()) {
            try {
                Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/shop", "root", "password");
                PreparedStatement stmt = conn.prepareStatement("SELECT item_id FROM cart WHERE user_email = ?");
                stmt.setString(1, email);
                ResultSet rs = stmt.executeQuery();

                while (rs.next()) {
                    JSONObject item = new JSONObject();
                    item.put("item_id", rs.getString("item_id"));
                    cartItems.put(item);
                }

                rs.close();
                stmt.close();
                conn.close();

            } catch (Exception e) {
                e.printStackTrace();
            }
        }
        
        response.getWriter().write(cartItems.toString());
    }
}

This servlet returns cart items as JSON, making it easy for the frontend to update dynamically.
