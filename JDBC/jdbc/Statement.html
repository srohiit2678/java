The Statement interface in Java is part of the JDBC (Java Database Connectivity) API and is used to execute SQL queries against a database.

Key Features of Statement Interface

  1. It is used for executing static SQL queries (queries that don’t require parameters).
  2. It is created using a Connection object.
  3. It provides methods to execute queries, update records, and retrieve results.

Creating a Statement

To use a Statement, follow these steps:

  1. Load and register the database driver.
  2. Establish a connection to the database.
  3. Create a Statement object.
  4. Execute SQL queries.
  5. Process the results (if any).
  6. Close the connection.

Example Code

java
import java.sql.*;

public class StatementExample {
    public static void main(String[] args) {
        // Database credentials
        String url = "jdbc:mysql://localhost:3306/your_database";
        String user = "your_username";
        String password = "your_password";

        try {
            // Step 1: Load JDBC Driver (not needed for newer JDBC versions)
            Class.forName("com.mysql.cj.jdbc.Driver");

            // Step 2: Establish a connection
            Connection conn = DriverManager.getConnection(url, user, password);

            // Step 3: Create a Statement
            Statement stmt = conn.createStatement();

            // Step 4: Execute a Query
            String sql = "SELECT * FROM students";
            ResultSet rs = stmt.executeQuery(sql);

            // Step 5: Process the Result Set
            while (rs.next()) {
                System.out.println("ID: " + rs.getInt("id") + ", Name: " + rs.getString("name"));
            }

            // Step 6: Close resources
            rs.close();
            stmt.close();
            conn.close();

        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Methods in Statement

MethodDescription
executeQuery(String sql)Used for SELECT queries. Returns a ResultSet.
executeUpdate(String sql)Used for INSERT, UPDATE, DELETE. Returns an int (number of affected rows).
execute(String sql)Used for general queries. Returns true if a ResultSet is returned, otherwise false.
close()Closes the statement and releases resources.

Drawbacks of Statement

  • SQL Injection Risk: Since Statement executes SQL queries as plain text, it is vulnerable to SQL injection.
  • Performance Issues: Repeated execution of similar queries leads to lower performance.

Better Alternative: PreparedStatement

If your queries include user inputs, PreparedStatement is recommended because:

  • It prevents SQL injection.
  • It improves performance by precompiling the query.

Batch Processing in JDBC (Statement Interface)

Batch processing in JDBC allows you to execute multiple SQL queries at once, reducing the number of interactions with the database. This improves performance and is useful when inserting or updating large amounts of data.

Why Use Batch Processing?

  • Reduces network traffic: Instead of sending multiple queries one by one, you send them all together.
  • Faster execution: The database processes all queries in a single go.
  • Efficient for bulk operations: Useful for inserting, updating, or deleting multiple records at once.

How to Perform Batch Processing?

Steps for Batch Processing using Statement

  1. Disable auto-commit mode (conn.setAutoCommit(false);).
  2. Create a Statement object.
  3. Add multiple SQL statements to the batch using addBatch(String sql).
  4. Execute the batch using executeBatch().
  5. Commit the transaction using conn.commit().
  6. Close the statement and connection.

Example: Batch Processing with Statement

java
import java.sql.*;

public class BatchProcessingExample {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/your_database";
        String user = "your_username";
        String password = "your_password";

        try {
            // Step 1: Load the driver (Optional for newer JDBC versions)
            Class.forName("com.mysql.cj.jdbc.Driver");

            // Step 2: Establish a connection
            Connection conn = DriverManager.getConnection(url, user, password);

            // Step 3: Disable auto-commit for batch processing
            conn.setAutoCommit(false);

            // Step 4: Create a Statement
            Statement stmt = conn.createStatement();

            // Step 5: Add multiple queries to the batch
            stmt.addBatch("INSERT INTO students (id, name, age) VALUES (101, 'Alice', 20)");
            stmt.addBatch("INSERT INTO students (id, name, age) VALUES (102, 'Bob', 22)");
            stmt.addBatch("INSERT INTO students (id, name, age) VALUES (103, 'Charlie', 21)");

            // Step 6: Execute the batch
            int[] results = stmt.executeBatch();

            // Step 7: Commit the transaction
            conn.commit();

            System.out.println("Batch executed successfully!");

            // Step 8: Close the statement and connection
            stmt.close();
            conn.close();

        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Explanation of the Code:

  1. Disabling Auto-commit (conn.setAutoCommit(false);)

    • This ensures that all queries are executed together as a single transaction.

  2. Adding Queries to the Batch

    • stmt.addBatch("INSERT INTO students ...") adds multiple queries to the batch.

  3. Executing the Batch

    • stmt.executeBatch(); executes all added queries at once.

  4. Committing the Transaction

    • conn.commit(); ensures that the changes are saved in the database.

Handling Errors in Batch Processing

If any query in the batch fails, you can use conn.rollback(); to undo all changes.

Example: Adding Error Handling

java
try {
    conn.setAutoCommit(false);
    Statement stmt = conn.createStatement();

    stmt.addBatch("INSERT INTO students (id, name, age) VALUES (101, 'Alice', 20)");
    stmt.addBatch("INSERT INTO students (id, name, age) VALUES (102, 'Bob', 22)");
    stmt.addBatch("INSERT INTO students (id, name, age) VALUES (103, 'Charlie', 21)");

    stmt.executeBatch();
    conn.commit();  // Commit changes if all queries succeed

} catch (SQLException e) {
    conn.rollback();  // Rollback changes if any error occurs
    e.printStackTrace();
}

Advantages of Batch Processing

✅ Faster Execution – Reduces communication between Java and the database.
✅ Efficient Resource Usage – Reduces overhead by executing multiple queries in one go.
✅ Useful for Bulk Operations – Ideal for inserting or updating thousands of records.

When to Use Batch Processing?

  • When inserting, updating, or deleting a large number of records.
  • When processing log files or importing data into a database.

Lesser-Known Facts About Statement in JDBC

While the Statement interface is commonly used in Java database operations, there are some interesting and lesser-known aspects of it. Here are a few:

1️⃣ Statement Can Execute Multiple SQL Statements at Once

Most people use Statement for a single SQL query at a time, but it can actually execute multiple statements separated by semicolons (;).

Example: Executing Multiple Queries

java
Statement stmt = conn.createStatement();
String sql = "INSERT INTO students (id, name) VALUES (201, 'Emma');"
           + "UPDATE students SET name = 'Emily' WHERE id = 201;"
           + "DELETE FROM students WHERE id = 202;";
stmt.execute(sql);  // Executes all three queries at once

💡 But be careful: Not all databases support multiple queries in one execute() call.

2️⃣ Fetching Large Data Sets Efficiently (setFetchSize)

When dealing with large amounts of data, retrieving the entire ResultSet at once can consume a lot of memory. You can optimize performance using setFetchSize(int rows), which tells JDBC how many rows to fetch at a time.

Example: Setting Fetch Size

java
Statement stmt = conn.createStatement();
stmt.setFetchSize(50);  // Fetches 50 rows at a time instead of all at once
ResultSet rs = stmt.executeQuery("SELECT * FROM students");

🚀 Why use it?

  • Saves memory when handling large datasets.
  • Improves performance by reducing network traffic.

3️⃣ Statement Can Retrieve Auto-Generated Keys

When inserting records into a table that has an auto-increment primary key, you can retrieve the generated key using getGeneratedKeys().

Example: Getting Auto-Generated Keys

java
Statement stmt = conn.createStatement();
stmt.executeUpdate("INSERT INTO students (name) VALUES ('Sophia')", Statement.RETURN_GENERATED_KEYS);

ResultSet rs = stmt.getGeneratedKeys();
if (rs.next()) {
    int generatedId = rs.getInt(1);
    System.out.println("New student ID: " + generatedId);
}

🔹 Use case: This is useful when inserting records where the primary key is generated by the database.

4️⃣ You Can Control the Number of Rows Returned (setMaxRows)

If you don’t want to retrieve all rows from a query, you can limit the number of results without modifying the SQL query.

Example: Limiting Results

java
Statement stmt = conn.createStatement();
stmt.setMaxRows(10);  // Limits the result set to 10 rows
ResultSet rs = stmt.executeQuery("SELECT * FROM students");

🔹 Advantage: This is useful when fetching only a preview of data without modifying the original query.

5️⃣ The Statement Object is Not Reusable for Different Queries

Unlike PreparedStatement, a Statement object cannot be reused for multiple different queries. If you try to use it after executing a query, you might get errors.

Example: Incorrect Usage

java
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM students"); // First query
stmt.executeUpdate("INSERT INTO students VALUES (301, 'Lucas')"); // Second query (Might cause an error)

🚨 Solution: Always create a new Statement object for each query or use PreparedStatement.

6️⃣ Statement is Slower Compared to PreparedStatement

  • Every time you execute a SQL query with Statement, the database parses and compiles it again.
  • With PreparedStatement, the query is precompiled and reused, making it much faster and safer (prevents SQL injection).

💡 Use PreparedStatement when: ✅ Queries contain user input (prevents SQL injection).
✅ The same query runs multiple times with different values.

Final Thoughts

✅ Statement is great for simple and static queries.
✅ Use setFetchSize() for handling large datasets.
✅ Retrieve auto-generated keys using getGeneratedKeys().
✅ Limit results using setMaxRows() instead of modifying SQL.
✅ For better performance and security, consider using PreparedStatement.

Is the Statement Object Reusable for Different Queries?

No, a Statement object should not be reused for different SQL queries. Once you execute a SQL query using Statement, you should create a new instance if you need to run another query.

Why Can't a Statement Object Be Reused?

  1. Each SQL query execution resets the Statement object.

    • If you try to execute a new query with the same Statement object, it might lead to unexpected behavior or errors.

  2. ResultSet Closes on a New Execution

    • If a Statement already has an open ResultSet, executing another query closes the previous ResultSet.
    • This makes it risky to use a single Statement for multiple queries in a loop.

Example: Incorrect Reuse of Statement

java
Statement stmt = conn.createStatement();
ResultSet rs1 = stmt.executeQuery("SELECT * FROM students");

ResultSet rs2 = stmt.executeQuery("SELECT * FROM teachers"); // Closes rs1

while (rs1.next()) {  // This will cause an error because rs1 is closed!
    System.out.println(rs1.getString("name"));
}

🚨 Problem: The second query execution closes rs1, causing an error when accessing it.

Correct Approach: Create a New Statement Object for Each Query

java
Statement stmt1 = conn.createStatement();
ResultSet rs1 = stmt1.executeQuery("SELECT * FROM students");

while (rs1.next()) {
    System.out.println(rs1.getString("name"));
}

Statement stmt2 = conn.createStatement();
ResultSet rs2 = stmt2.executeQuery("SELECT * FROM teachers");

while (rs2.next()) {
    System.out.println(rs2.getString("name"));
}

// Close statements after use
stmt1.close();
stmt2.close();

✅ Solution: Use separate Statement objects for different queries.

What About PreparedStatement?

Unlike Statement, PreparedStatement can be reused with different parameters.

Example: Reusable PreparedStatement

java
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM students WHERE age > ?");
pstmt.setInt(1, 18);
ResultSet rs1 = pstmt.executeQuery();  // First execution

pstmt.setInt(1, 21);
ResultSet rs2 = pstmt.executeQuery();  // Second execution with different value

✅ Better performance because it is compiled once and executed multiple times.
✅ Prevents SQL injection by using placeholders (?).

Summary

FeatureStatementPreparedStatement
Reusable for multiple queries?❌ No✅ Yes
Closes ResultSet on new execution?✅ Yes❌ No
Performance for repeated queries🔴 Slower🟢 Faster
Prevents SQL Injection?❌ No✅ Yes

So, for simple one-time queries, Statement is fine. But for repeated queries or user inputs, use PreparedStatement!

Types of Statements in Java JDBC

In Java JDBC, there are three main types of statements used for executing SQL queries:

Statement TypeDescription
StatementUsed for simple SQL queries without parameters.
PreparedStatementPrecompiled SQL statement that improves performance and security (prevents SQL injection).
CallableStatementUsed to call stored procedures in a database.

1️⃣ Statement (Simple & Static Queries)

  • Used for executing simple SQL queries.
  • Cannot accept parameters dynamically.
  • Slower because the SQL query is compiled every time it is executed.
  • Not safe from SQL injection.

Example: Using Statement

java
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery("SELECT * FROM students");

🚨 Downside: If user input is involved, this can be vulnerable to SQL injection.

2️⃣ PreparedStatement (Parameterized Queries)

  • Used for executing queries with parameters (?).
  • Precompiled once and executed multiple times with different values.
  • Faster than Statement when executing queries repeatedly.
  • Safe from SQL injection.

Example: Using PreparedStatement

java
PreparedStatement pstmt = conn.prepareStatement("SELECT * FROM students WHERE age > ?");
pstmt.setInt(1, 18);  // Setting parameter
ResultSet rs = pstmt.executeQuery();

✅ Better performance & security because the SQL is precompiled.

3️⃣ CallableStatement (Stored Procedures)

  • Used to call stored procedures in the database.
  • Supports IN, OUT, and INOUT parameters.
  • Improves performance by executing pre-defined SQL logic.

Example: Calling a Stored Procedure

java
CallableStatement cstmt = conn.prepareCall("{CALL getStudentById(?)}");
cstmt.setInt(1, 101);
ResultSet rs = cstmt.executeQuery();

✅ Efficient for executing complex business logic stored in the database.

Which One Should You Use?

FeatureStatementPreparedStatementCallableStatement
Use CaseSimple queriesQueries with parametersCalling stored procedures
PerformanceSlow (recompiled each time)Fast (precompiled)Fast (uses stored procedures)
SQL Injection Protection❌ No✅ Yes✅ Yes
Parameter Support❌ No✅ Yes (? placeholders)✅ Yes (IN/OUT params)
Best ForStatic queriesDynamic queriesComplex logic stored in DB

Final Thoughts

  • Use Statement for simple queries without parameters.
  • Use PreparedStatement when queries involve user input (safer & faster).
  • Use CallableStatement if you need to call stored procedures.
